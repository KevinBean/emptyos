# Code of Conduct

EmptyOS is built on the premise that a system should augment human judgment, not replace it. The same principle applies to the people building and using it: we work *with* each other, not *at* each other.

## The short version

- Be kind. Assume good faith.
- Critique ideas, not people.
- Disagree freely; do it without contempt.
- If you're unsure whether something crosses a line, it probably does.

## Expected behaviour

- Welcoming language toward newcomers — everyone was once someone who didn't know how the codebase worked.
- Constructive feedback that points at the code, the design, or the decision — never at the person who wrote it.
- Patience with questions, especially the basic ones. The basic ones are how the project gets clearer.
- Crediting prior work and other contributors honestly.

## Unacceptable behaviour

- Personal attacks, harassment, sustained dismissiveness, or contempt directed at a person.
- Discrimination based on identity (race, gender, sexuality, nationality, religion, disability, age, appearance, neurotype, experience level).
- Sexualised language, unwanted advances, or sexual imagery in project spaces.
- Doxxing, sharing others' private information, or threats of any kind.
- Repeatedly disrupting discussions, ignoring moderation, or trying to litigate moderation decisions in public threads.

## Scope

This applies to all project spaces — the GitHub repo (issues, PRs, discussions), any official chat or social presence, and any time someone is representing the project in public.

## Reporting

If you see or experience behaviour that violates this code, please email **conduct@binbian.net**. Reports stay confidential. Include what happened, where, and any context that helps.

If the report concerns the maintainer, escalate via GitHub by emailing GitHub Support or by raising it publicly in a way you're comfortable with.

## Enforcement

The maintainer will review reports and respond with one of:

1. **Clarification** — a private conversation explaining what was off and why.
2. **Warning** — a documented note that the behaviour needs to stop.
3. **Temporary ban** — exclusion from project spaces for a defined period.
4. **Permanent ban** — for severe or repeated violations.

There is no appeals court. The maintainer's call stands. If you disagree, you're free to fork — that's how open source works.
